📊 WHAT THIS TEST DOES
=======================

TEST OBJECTIVE:
Validate SDA fabric resilience when the ENTIRE Port-Channel between
L2 Border (FS2_L2H-1) and Border Controller (FS2_BC1) fails.

KEY DIFFERENCE FROM TC 07-01:
- TC 07-01: Shutdown ONE member → Po40 stays UP (hitless)
- TC 07-02: Shutdown BOTH members → Po40 goes DOWN (traffic loss)

EXPECTED BEHAVIOR:
✅ Po40 goes DOWN (protocol down) - EXPECTED
✅ OSPF adjacency to BC1 goes DOWN - EXPECTED
✅ LISP session to BC1 goes DOWN - EXPECTED
✅ Packet loss during convergence (3-10 seconds) - EXPECTED
✅ Dead flows detected - EXPECTED
✅ Full recovery after restoration - REQUIRED
✅ Convergence times measured via Spirent PLA - REQUIRED

📋 TEST PHASES
==============

PHASE 1: BASELINE (5-10 minutes)
--------------------------------
- Verify Spirent 0.000% loss (GATE)
- Catalyst Center health check
- Collect CLI from L2H-1 and BC1
- 3 screenshots (Spirent, CC health, CC network health)

PHASE 2: FAILURE (10-15 minutes)
---------------------------------
- Shutdown BOTH Te4/0/1 AND Te4/0/2 on L2H-1
- Po40 goes DOWN (expected)
- Monitor Spirent convergence (3 minutes)
- STOP traffic, export .tcc DB
- Upload to Spirent PLA for analysis
- Download PLA Excel with convergence data
- Collect CLI during failure
- 3 screenshots + PLA analysis

PHASE 3: RECOVERY (10-15 minutes)
----------------------------------
- Restore both members (no shutdown)
- Monitor Po40 rebundle (10-30 sec)
- Monitor OSPF convergence (15-30 sec)
- Monitor LISP session restore (10-30 sec)
- Restart Spirent, verify 0.000%
- Collect CLI post-recovery
- 3 screenshots

TOTAL TIME PER ITERATION: 25-40 minutes
TOTAL TIME (3 iterations): 75-120 minutes (1.5-2 hours)

🔧 TROUBLESHOOTING
==================

Problem: Script fails with "No module named 'netmiko'"
Solution: pip install netmiko

Problem: Cannot SSH to devices
Solution: Verify VPN connected, check IP addresses in ssh.morgan.stanley.svs.sda.csv

Problem: Po40 fails to rebundle after recovery
Solution: Check "show lacp 40 counters" and "show lacp 40 neighbor"
          Verify physical links: "show interfaces Te4/0/1" and "Te4/0/2"
          Check BC1 side: ssh admin1@172.31.2.0, "show etherchannel 40 summary"

Problem: OSPF doesn't converge
Solution: Check "show ip ospf neighbor detail"
          Verify Po40 is UP: "show interfaces Po40"
          Check for authentication issues: "show logging | include OSPF"

Problem: LISP sessions don't restore
Solution: Check "show lisp session" and "show lisp session verbose"
          Verify routing: "show ip route 192.168.100.1"
          Ping BC1/BC2: "ping 192.168.100.1 source lo0"

Problem: Spirent traffic doesn't restore
Solution: Wait 60 seconds after full convergence
          Check routing: "show ip route summary"
          Check CTS: "show cts role-based counters" (look for denies)
          Verify VXLAN: "show lisp instance-id * ethernet server"

🎯 EXPECTED CONVERGENCE TIMES
==============================

Based on similar tests and industry best practices:

| Metric                   | Expected Range  | Notes                    |
|--------------------------|-----------------|--------------------------|
| Po40 Down Detection      | Immediate       | LACP detects no members  |
| OSPF Adjacency Down      | 1-10 seconds    | Dead timer or BFD        |
| Packet Loss Duration     | 3-10 seconds    | Until reconvergence      |
| Dead Flows               | 50-200          | Depends on traffic       |
| LACP Rebundle Time       | 10-30 seconds   | After no shutdown        |
| OSPF Convergence to FULL | 15-30 seconds   | Hello + adjacency form   |
| LISP Session Restore     | 10-30 seconds   | TCP 3-way handshake      |
| Total Recovery Time      | 30-60 seconds   | All protocols converged  |

If your results are significantly outside these ranges, investigate:
- LACP timers (slow vs. fast)
- OSPF timers (Hello, Dead)
- BFD configuration (may speed up detection)
- Network load (high load may slow convergence)
- Hardware performance (ASIC, CPU)
